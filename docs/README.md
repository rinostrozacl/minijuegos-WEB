# Documentación: Plataforma de Competencia de Minijuegos

Este directorio contiene la documentación completa para la plataforma web de la competencia de minijuegos con temáticas chilenas.

## Estructura de la Documentación

La documentación está organizada en las siguientes carpetas:

### [requerimientos/](./requerimientos/)
Contiene los documentos relacionados con los requerimientos funcionales y no funcionales del sistema:
- **REQUERIMIENTOS_GENERALES.md**: Introducción, alcance, descripción general y listado de requerimientos.
- **CASOS_USO.md**: Descripción detallada de los casos de uso principales del sistema.

### [implementacion/](./implementacion/)
Contiene los documentos relacionados con la implementación técnica:
- **ARQUITECTURA.md**: Arquitectura técnica, componentes principales y modelo de datos.
- **PLAN_IMPLEMENTACION.md**: Fases de desarrollo, cronograma y consideraciones para la implementación.
- **CREDENCIALES_FIREBASE.md**: Configuración de Firebase y gestión de credenciales.

### [modulos/](./modulos/)
Contiene la documentación detallada de cada módulo funcional:
- **AUTENTICACION.md**: Módulo de autenticación, registro y verificación de correo.
- **REGISTRO_PARTICIPANTES.md**: Módulo de registro de participantes y selección de temáticas.
- **GESTION_JUEGOS.md**: Módulo de gestión y visualización de juegos.
- **EVALUACION.md**: Módulo de evaluación de juegos.
- **NOTIFICACIONES.md**: Módulo de notificaciones por correo.
- **ADMINISTRACION.md**: Módulo de administración y preregistro masivo.

## Cómo Utilizar Esta Documentación

1. Comienza con los documentos en la carpeta **requerimientos/** para obtener una visión general del proyecto.
2. Consulta los documentos en la carpeta **modulos/** para obtener detalles de implementación de cada funcionalidad.
3. Revisa los documentos en la carpeta **implementacion/** para entender la arquitectura técnica y el plan de desarrollo.
