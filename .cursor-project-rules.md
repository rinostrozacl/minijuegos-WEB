# Reglas de Proyecto para Cursor - EVA Videojuegos

Este documento establece reglas y convenciones específicas para este proyecto, con el fin de optimizar el trabajo con la asistencia de IA en Cursor.

## Estructura y Navegación

- **Directorio Principal**: `/Users/rinostroza/Documents/code/EVA Videojuegos`
- **Aplicación Nuxt**: `/Users/rinostroza/Documents/code/EVA Videojuegos/Plataforma Web/app`
- **Documentación**: `/Users/rinostroza/Documents/code/EVA Videojuegos/Plataforma Web/docs`

## Tecnologías y Frameworks

- **Frontend**: Nuxt 3 con Vue 3 (Composition API)
- **UI Framework**: Nuxt UI / Tailwind CSS
- **Backend**: Firebase (Firestore, Authentication, Storage, Cloud Functions)
- **Idioma de Desarrollo**: Español para comentarios, documentación y variables

## Convenciones de Código

1. **Nombrado de Componentes**:

   - **Componentes Vue**: PascalCase para nombres (ej. `LoginForm.vue`)
   - **Composables**: camelCase con prefijo 'use' (ej. `useAuth`)
   - **Modelos**: Sustantivo singular (ej. `Game`, `User`)

2. **Estructura de Archivos Nuxt**:

   - Componentes en directorio `/components`
   - Páginas en directorio `/pages`
   - Layouts en directorio `/layouts`
   - Composables en directorio `/composables`
   - Middleware en directorio `/middleware`
   - API endpoints en directorio `/server/api`

3. **CSS/Tailwind**:

   - Utilizar clases de Tailwind CSS para estilos
   - Componentes de Nuxt UI para elementos de interfaz comunes
   - Mantener theme personalizado en `/assets/css`

4. **TypeScript**:
   - Interfaces para modelos de datos
   - Tipo explícito para todas las variables
   - Comentarios JSDoc para funciones públicas

## Gestión de Estado

- Usar Pinia para gestión de estado global
- Composables para lógica reutilizable y estado local
- Reactivity API de Vue 3 (ref, reactive, computed, watch)

## Convenciones de Archivo

- **Nombres de archivos**: kebab-case para archivos de utilidad
- **Nombres de componentes**: PascalCase.vue
- **Imports**: Importación automática con resolución de Nuxt

## Firebase

- **Colecciones Firestore**: Nombres en inglés, plural
  - `users`, `games`, `evaluations`, etc.
- **Referencias en código**: Utilizar constantes para nombres de colecciones
- **Storage**: Estructura de carpetas según tipo de archivo
  - `/games/{gameId}/...`
  - `/users/{userId}/...`

## Documentación Interna

- Comentarios en español
- Documentar decisiones de arquitectura
- JSDoc para interfaces y funciones públicas

## Desarrollo y Pruebas

- Tests unitarios con Vitest
- End-to-end testing con Playwright (opcional)
- Enfoque Mobile-First para interfaces
- Validación de formularios con vee-validate o zod

## IA y Automatización

- **Carpeta de Contexto IA**: `.ia/` (debe estar en gitignore)

  - Obligatoria en la raíz del proyecto.
  - Archivos requeridos:
    - `.ia/historial.md` (historial cronológico de desarrollo)
    - `.ia/proyectos.json` (metadata de proyectos activos)
    - `.ia/preferencias.json` (preferencias de desarrollo)
    - `.ia/estadisticas.json` (métricas e interacciones)
  - Si algún archivo falta, debe crearse automáticamente al iniciar una nueva sesión.
  - El equipo debe consultar y actualizar estos archivos al realizar cambios importantes en arquitectura, dependencias o flujos clave.
  - Estructura y uso detallados en las **Reglas del Sistema** (ver documentación interna o instrucciones personalizadas de IA).

- **Etiquetas de Contexto**:
  - `#auth` - Autenticación y gestión de usuarios
  - `#games` - Gestión de juegos y temáticas
  - `#eval` - Sistema de evaluación
  - `#admin` - Panel administrativo
  - `#ui` - Componentes de interfaz

## Procedimientos Específicos

1. **Creación de Componentes**:

   ```
   mkdir -p components/[module]/[ComponentName] && touch components/[module]/[ComponentName]/index.vue
   ```

2. **Creación de Páginas**:

   ```
   touch pages/[route].vue
   ```

3. **Compilación y Despliegue**:
   ```
   pnpm build
   firebase deploy
   ```
